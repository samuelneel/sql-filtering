# SQL: Apply Filters to Investigate Security Data

A SQL project investigating security issues in a hypothetical organization by filtering login-attempt logs and employee records.

## Skills Demonstrated
- Writing `SELECT` queries with `WHERE` clauses to filter security-relevant records
- Combining conditions with logical operators (`AND`, `OR`, `NOT`)
- Pattern matching with `LIKE` and wildcards (`%`)
- Investigating login anomalies and reviewing access by department and location

## Summary
Using the `log_in_attempts` and `employees` tables, I filtered records to surface after-hours failed logins, activity on specific dates, and employee access by department and location.
